web Security

knowledge required

SQL Injectioin

    • union select *(全部列名) from admin(表名) where id(字段) = 1;
  • error
    • 1
  • Blind
    • 1
  • Stacked

vulnerability cause

  • 攻击发生在不可信的数据作为命令或者查询语句的一部分,被发送给解释器的时候.攻击者发送的恶意数据可以欺骗解释器,以执行计划外的命令或者在未被恰当授权时访问数据.

Protective measures

  • 使用安全的API,完全避免使用解释器或提供参数化界面的API.
  • escape类过滤避免特殊字符.
  • 黑/白名单机制.

  • 业务接口没有做验证或机制缺陷,枚举
